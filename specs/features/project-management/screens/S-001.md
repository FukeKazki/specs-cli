# Screen: リストビュー画面

## 目的

要件を表形式で俯瞰し、状態や期限を一覧で把握する画面（PM 目線）。

## 項目

- 要件（要件の H1。R-00n を含む）
- priority（Must / Should / Could / Won't）
- feature 名
- status（未着手 / 進行中 / レビュー / 完了）
- assignee（担当）
- start / due（開始日 / 期限日）

## 操作

- [x] A-001 列によるソート
- [x] A-002 feature・priority・status・担当でのフィルタ
- [x] A-003 行を選択して詳細（右サイドパネル / ドキュメント）へ遷移する
- [x] A-004 status / assignee / start / due を編集して frontmatter に保存する

## エラー

- 一覧取得失敗時はトーストでメッセージを表示する
- 要件が 0 件の場合は「要件がありません」を表示する
